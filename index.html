Excana

Isometric action game with an emphasis on overwhelming numbers and unbalanced odds. The player must navigate a series of arenas filled with hordes of enemies. Subsequent areas add new enemy types and a new weapon for the player to use. The player's weapons attack on their own but can unleash a powerful ability by using charge gained from killing enemies.

This project was created for Abertay's Professional Project module where students from several discplines working as a team to create a prototype based on a brief from an external client. With one other programmer on the team I developed all of the player driven functionality while they developed the enemy and level functionality.

Throughout the module we used several tools like Jira and Perforce to simulate what it could be like in a real work environment. We also adhered to the agile development methodology by splitting development into sprints and performing stand up, planning and retrospective meetings.

Core Contributions:
  • UI: Player HUD and menu functionality.
  • Camera hiding terrain that obstructs view of the player.
  • Create a weapon template that allows designers to easily add more weapons
  • Player auto attack, special attack and aiming functionality.
  • Simple saving and loading between levels.
  • Cosmetic effects such as hitstop.

Development Time: 4 months

Team Size: 8

Tools Used: Unreal Engine 5, Blueprint and C++

DirectX 11 Scene

A piece of coursework that demonstrates an understanding of several graphics programming techniques. Every technique in the scene has several parameters that can be customised such as the total height of the vertex manipulation.

List of Features:
  • Sine wave ocean and height map terrain.
  • Up to 8 lights with working shadows. Light can either be a directional light, point light or spotlight.
  • Depth of field with optimised Gaussian blur.
  • Tesselation based on camera position.

Development Time: 1 month

Team Size: Solo

Built using: DirectX 11, C++, HLSL

Unreal Behaviour Tree

Coursework created to demonstrate how behaviour for guards in a stealth game can be created through the use of behaviour trees. This project uses a custom behaviour tree implementation as opposed to Unreal's built-in solution.


Main Behvaiour:
  • Patrol a set path, stopping at locations set by the level creator.
  • If a player is seen for a set time, investigate their last seen position
  • If the player is seen again while investigating, chase after them until line of sight is lost

Development Time: 3 Weeks

Team Size: Solo

Built using: Unreal Engine 5

Grappling Hook Mechanic

Grappling hook mechanic heavily inspired by Attack on Titan.


Main Features:
  • Shoot and swing from multiple grappling hooks.
  • Swing faster or reel in using a "thruster" attached to the character.
  • Slow down time to aim better when moving quickly.
  • Limit thrust capacity with a gas meter.
  • Gas can be refilled by picking up collectables.

Development Time: 2 months

Team Size: Solo

Built using: Unreal engine 5, blueprint and C++
